In `_handle_upload()`, after the `classify_dataframe` call and BEFORE the DRR column is added, insert the LLM classification block:

```python
# Run classification pipeline
registry = RuleRegistry(build_default_rules())
df = classify_dataframe(df, registry)

# --- LLM Classification Fallback ---
llm_cfg = LLMConfig()
if llm_cfg.enabled:
    ui.notify(t("llm.classifying"), type="info")
    drr_table_for_llm = DRRTable.from_csv(DRR_CSV_PATH)
    vm_records: list[dict[str, Any]] = df.to_dict(orient="records")
    vm_records = await classify_unknown_vms_async(vm_records, drr_table_for_llm, llm_cfg)
    df = pd.DataFrame(vm_records)
    llm_count = sum(1 for r in vm_records if r.get("classification_confidence") == "llm")
    if llm_count > 0:
        ui.notify(t("llm.classified_notify", count=llm_count), type="positive")
# --- End LLM Classification Fallback ---

# Add DRR column from reference table
drr_table = DRRTable.from_csv(DRR_CSV_PATH)
```

Add `from typing import Any` import if not already present (check existing imports first — it may already be there from pandas stubs usage).

The existing DRR lookup after the LLM block uses `drr_table` — keep it as-is. The LLM block uses a separate `drr_table_for_llm` instance (same CSV, loaded once for valid_categories extraction inside classify_unknown_vms_async).
