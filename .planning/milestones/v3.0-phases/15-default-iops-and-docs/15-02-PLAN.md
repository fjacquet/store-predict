**ADR-059** (`docs/adr/059-default-iops-estimates.md`):
Follow exact ADR format from existing ADRs (see docs/adr/index.md pattern). Content:

- **Title:** ADR-059: Workload-based IOPS defaults for RVTools sizing
- **Date:** 2026-02-21
- **Status:** Accepted
- **Context:** RVTools exports contain no performance data (IOPS, throughput). When layout engine processes RVTools imports, all VMCalculation objects have peak_iops=0, making the IOPS budget constraint inactive. This produces unrealistic layouts ignoring I/O contention. Pre-sales sizing requires defensible IOPS estimates even without measured data.
- **Decision:** When `CalculationSummary.has_performance_data` is False, apply workload-based IOPS estimates via `_apply_default_iops()` before layout placement. Values loaded from `samples/IOPS.csv` (semicolon-delimited), falling back to hardcoded constants if file is missing. Estimates are conservative (peak IOPS, not average). `avg_iops` estimated as 70% of peak. Linux/Windows IOPS split (40 vs 50) is NOT implemented — data model has no OS distinction at workload_category level; 50 IOPS used for all generic VMs (documented limitation).
- **Consequences:** (+) Realistic layouts for RVTools imports, (+) pre-sales can adjust IOPS via CSV, (+) conservative bias prevents under-provisioning, (-) estimates may over-provision for low-I/O workloads, (-) no OS-level IOPS differentiation possible without data model change.
