  <action>
    Create `src/store_predict/ui/pages/layout_page.py` following the `report.py` page structure.

    **Page skeleton:**
    - `@ui.page("/layout")` async function `layout_page()`
    - `await ui.context.client.connected()` first
    - Guard: check `app.storage.tab.get("vm_data")` — if None, show empty-state card with icon "grid_view", message from `t("layout_page.no_data")`, and button to navigate to `/upload`. Use same card pattern as report.py guard. Then `return`.
    - Load constraints via `_load_constraints()` helper
    - Call `calculate(vm_data)` then `generate_all_proposals(summary, constraints)`
    - Render inside `with layout("StorePredict - Layout"), ui.column().classes("w-full p-4 gap-4"):`

    **_load_constraints() / _save_constraints() helpers:**
    - Read/write 5 constraint values from/to `app.storage.tab` with `layout_` prefix keys
    - Keys: `layout_max_ds_mib` (int, default 4*1024*1024), `layout_max_vms` (int, default 25), `layout_iops_budget` (float, default 100_000.0), `layout_snapshot_pct` (float, default 15.0), `layout_growth_pct` (float, default 20.0)
    - Return `PlacementConstraints(...)` from loaded values
    - PlacementConstraints is frozen — always create new instances

    **Advanced Settings panel (REQ-007):**
    - `ui.expansion(t("layout_page.settings_title"), icon="settings", caption=t("layout_page.settings_subtitle")).classes("w-full border border-gray-200 rounded-lg")`
    - Inside the expansion, 5 controls:
      1. Max DS capacity: `ui.select(options={2*1024*1024: "2 TB", 4*1024*1024: "4 TB", 8*1024*1024: "8 TB", 16*1024*1024: "16 TB", 32*1024*1024: "32 TB", 64*1024*1024: "64 TB"}, ...)` with label from `t("layout_page.max_ds_capacity")`
      2. Max VMs/DS: `ui.slider(min=5, max=50, step=1, ...)` with label from `t("layout_page.max_vms_per_ds")` — display current value with `ui.label().bind_text_from(slider, "value")`
      3. IOPS budget: `ui.number(label=t("layout_page.iops_budget"), min=10_000, max=1_000_000, step=10_000, ...)`
      4. Snapshot reserve %: `ui.slider(min=0, max=30, step=1, ...)` with label `t("layout_page.snapshot_reserve")`
      5. Growth margin %: `ui.slider(min=0, max=40, step=1, ...)` with label `t("layout_page.growth_margin")`
    - Each control's `on_change` calls `_on_settings_change()` which saves to `app.storage.tab`, rebuilds constraints, and calls `_rebuild_layout()`
    - For sliders, add `.props('label-always')` or use throttle to avoid excessive re-generation. Use `on_change` with a simple approach — the datasets are small enough that re-generation is fast.

    **Comparison table (REQ-008):**
    - Create `_build_comparison_table(proposals: list[LayoutProposal])` function
    - `ui.table` with columns: Metric, Consolidation, Performance, Uniform
    - Rows from LayoutMetrics fields: ds_count, raw_capacity (formatted as TiB), usable_capacity, used_capacity, avg_utilization, min_utilization, max_utilization, avg_vm_density, max_vm_density, total_iops, max_iops_single_ds, iops_headroom, isolation_score, snapshot_granularity_rating, oversized_vm_count
    - All metric labels via `t("metrics.XXX")` keys
    - Add `_fmt_tib(mib: float) -> str` helper: `f"{mib / (1024 * 1024):.2f} TiB"`

    **Recommended strategy indicator (REQ-008):**
    - Create `_recommend_strategy(proposals: list[LayoutProposal]) -> str` function
    - Logic: if highest isolation_score > 0.5 → "performance"; if single workload category across all VMs → "consolidation"; else → "uniform". Ties broken by fewest datastores.
    - Display as `ui.badge(t("layout_page.recommended"), color="green")` next to the recommended strategy column header, or as a colored star icon row in the comparison table

    **Reactive rebuild (REQ-007):**
    - `results_container = ui.column().classes("w-full")` — holds comparison table + placeholder for detail tabs (Plan 16-02)
    - `_rebuild_layout(container, vm_data, constraints)`: calls `calculate()` + `generate_all_proposals()`, then `container.clear()` and re-renders comparison table inside container
    - Wire the results_container into the settings panel callbacks

    **Navigation (REQ-010):**
    - In `src/store_predict/ui/layout.py`, add Layout link after the Report link: `ui.link(t("layout.layout"), "/layout").classes("text-white no-underline hover:underline")`
    - In `src/store_predict/main.py`, add import: `import store_predict.ui.pages.layout_page  # noqa: F401`

    **i18n keys:**
    - Add ALL required keys to both `en.yaml` and `fr.yaml` (~35-40 keys). See the research RESEARCH.md "i18n Keys Required" section for the full list. Include sections: `layout.layout`, `layout_page.*`, `strategy.*`, `metrics.*`, `ds.*`.
    - French translations: use professional technical French (e.g., "Disposition des datastores", "Paramètres avancés", "Capacité maximale du datastore", "Score d'isolation", etc.)

    **Anti-patterns to avoid:**
    - Do NOT name this file `layout.py` — it conflicts with `ui/layout.py`. Use `layout_page.py`.
    - Do NOT use f-strings for any Vue template content (add_slot). Use raw strings `r'''...'''`.
    - Do NOT mutate PlacementConstraints — always create new instances from tab storage.
    - Do NOT use `ui.aggrid` — use `ui.table` only.
    - Do NOT use loop variable `t` — it shadows the i18n `t()` import. Use `wt` or `tab_item`.
