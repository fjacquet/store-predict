    - Use `ui.tabs()` with 3 tabs: Consolidation (icon "compress"), Performance (icon "speed"), Uniform (icon "balance")
    - Tab labels from `t("strategy.consolidation")`, `t("strategy.performance")`, `t("strategy.uniform")`
    - Use `ui.tab_panels(tabs, value=first_tab).classes("w-full")` to wrap 3 `ui.tab_panel` sections
    - Each panel calls `_build_strategy_detail(proposal)` for its respective proposal
    - Do NOT use loop variable `t` for tab — use `tab_consol`, `tab_perf`, `tab_unif` or similar names to avoid shadowing the i18n `t()` import

    **Per-strategy detail (`_build_strategy_detail`):**
    - Show strategy description label: `t("strategy.consolidation_desc")` etc.
    - Show summary stats row: total DS count, total capacity, avg utilization as `ui.label` badges
    - Call `_build_datastore_table(proposal.datastores)` for the expandable table

    **Expandable datastore table (`_build_datastore_table`):**
    - `ui.table` with columns: expand button (auto-width), Name, Raw Capacity, Used, Utilization %, VM Count, IOPS, Workload Types
    - Column definitions use `t("ds.name")`, `t("ds.raw_cap")`, etc. for labels
    - Row data built from `DatastoreRecommendation` fields:
      - `name`: ds.name
      - `raw_cap`: `_fmt_tib(ds.raw_capacity_mib)` (reuse helper from Plan 16-01)
      - `used`: `_fmt_tib(ds.used_capacity_mib)`
      - `util_pct`: `f"{ds.utilization_pct:.1f}%"`
      - `util_pct_raw`: `ds.utilization_pct` (numeric for color logic in template)
      - `vm_count`: `ds.vm_count`
      - `iops`: `f"{ds.total_iops:,.0f}"`
      - `workloads`: `", ".join(sorted(ds.workload_types))`
      - `vm_names`: `[vm.vm_name for vm in ds.assigned_vms]` (list for expand drill-down)
    - Set `row_key="name"` on the table

    **Expandable row via Quasar slots:**
    - Use `table.add_slot('header', ...)` with raw string template that adds an empty `<q-th auto-width />` before the column headers
    - Use `table.add_slot('body', ...)` with raw string template containing:
      1. Main row: `<q-tr :props="props">` with expand button `<q-btn @click="props.expand = !props.expand" :icon="props.expand ? 'expand_less' : 'expand_more'" size="sm" color="primary" round dense />` in first `<q-td auto-width>`, then `<q-td v-for="col in props.cols" :key="col.name" :props="props">{{ col.value }}</q-td>`
      2. Expanded row: `<q-tr v-show="props.expand" :props="props"><q-td colspan="100%" class="bg-gray-50">` containing a div with "VMs assigned:" heading and `v-for="vm in props.row.vm_names"` to list each VM name
    - ALL template content MUST use raw strings `r'''...'''` — NEVER f-strings (Vue `{{ }}` syntax conflicts with Python f-string `{}`)

    **Utilization color-coding:**
    - Use `table.add_slot('body-cell-util_pct', ...)` with raw string template that applies Tailwind classes based on `props.row.util_pct_raw`:
      - `> 80` → `text-red-600 font-bold`
      - `> 60` → `text-yellow-600`
      - else → `text-green-600`
    - NOTE: If using a full body slot (for expand), the util color logic must be embedded in the body slot template directly using a ternary class binding on the util_pct `<q-td>`. Do NOT add a separate body-cell slot if you already have a full body slot.

    **Integration with reactive rebuild:**
    - Update the existing `_rebuild_layout()` function (from Plan 16-01) to also call `_build_strategy_tabs(proposals)` inside the results_container, after the comparison table
    - When settings change, both comparison table AND detail tabs rebuild together

    **Handle empty datastores:**
    - If a proposal has 0 datastores, show `ui.label(t("layout_page.no_datastores")).classes("text-gray-400 italic p-4")` instead of the table

    **Anti-patterns to avoid:**
    - Do NOT use `ui.aggrid` — AG Grid master-detail is enterprise-only. Use `ui.table` with `add_slot`.
    - Do NOT use `ui.teleport` for expanded rows — known DOM rebuilding issues. Use `props.expand` in body slot.
    - Do NOT use f-strings for Vue templates — use `r'''...'''` raw strings.
    - Do NOT use loop variable `t` anywhere — shadows i18n import.
