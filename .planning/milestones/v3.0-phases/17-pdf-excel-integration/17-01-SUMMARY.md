# Phase 17 Plan 01: PDF & Excel Layout Integration Summary

**One-liner:** Layout recommendations table added to PDF page 3 and Excel sheet 4 using shared 15-metric _layout_metric_rows helper with FR/EN locale support.

## What Was Built

This plan extends both export services to include datastore layout strategy comparison output generated by `pipeline/layout_engine.generate_all_proposals()`.

### PDF Report (pdf_report.py)

- Added `_fmt_tib()` and `_fmt_pct()` one-liner formatting helpers
- Added `_layout_metric_rows(proposals)` shared builder — returns 15-metric tuples for all 3 strategies; uses dict lookup by `strategy_name` to avoid index assumptions; exported in `__all__`
- Added layout page to `generate_report_pdf()` after the charts section (Page 3): `PageBreak()` + heading + 4-column `Table` (Metric | Consolidation | Performance | Uniform) with `_BRAND_BLUE` header, Vera fonts, 9pt, grey grid, RIGHT-aligned values
- Guard: `if summary.total_vms == 0` skips the layout page entirely

### Excel Report (excel_report.py)

- Added `_write_layout_sheet()` helper with strategy comparison summary at top (16 rows: header + 15 metrics) followed by 3 per-strategy datastore detail sub-tables
- Each sub-table has: strategy heading, DS column headers (7 columns), data rows with TiB capacity, utilization %, VM count, IOPS, workload types
- `autofit()` and `freeze_panes(1, 0)` applied
- Wired into `generate_report_xlsx()` before `wb.close()`
- Guard: `if summary.total_vms == 0: return` skips sheet (produces 3-sheet workbook)
- Imports `_layout_metric_rows` from `pdf_report` to share metric formatting logic
