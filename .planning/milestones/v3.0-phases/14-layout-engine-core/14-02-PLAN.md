  <action>
Add to the existing `layout_engine.py` (which already has consolidation, BFD, metrics, default IOPS from Plan 14-01):

1. **Performance strategy tier classification:**

   Define a `PerformanceTier` StrEnum: `HOT`, `WARM`, `COLD`.

   Define `_ISOLATION_WORKLOADS` and thresholds as module-level constants:
   - `_ISOLATION_CAPACITY_THRESHOLD_MIB = 2 * 1024 * 1024` (2 TiB)
   - `_ISOLATION_IOPS_THRESHOLD = 5_000.0`

   `_is_mission_critical(vm) -> bool`: True if workload contains "SAP HANA" or "Exchange", OR `required_mib > threshold`, OR `peak_iops > threshold`. Use substring matching with `in`.

   `_classify_tier(vm) -> PerformanceTier`:
   - HOT: `peak_iops > 500` or workload contains "Database"
   - WARM: `100 <= peak_iops <= 500`
   - COLD: otherwise

2. **`_isolate_vms(vms, constraints) -> tuple[list[_DatastoreBuilder], list[VMCalculation]]`:**
   - Separate mission-critical VMs into 1:1 dedicated datastores
   - Size each isolated DS to the VM: `raw_mib = vm.required_mib / constraints.usable_ratio`
   - Naming by workload type: DS_HANA_NN, DS_EXCHANGE_NN, DS_ORA_NN, DS_ISOLATED_NN
   - Use a `dict[str, int]` for per-prefix counters (naming counter independence per Pitfall 5)
   - Return (isolated_bins, remaining_vms)

3. **`_performance_strategy(vms, constraints) -> LayoutProposal`:**
   - Phase 0: `_isolate_vms(vms, constraints)` to extract mission-critical VMs
   - Phase 1: Classify remaining into tiers using `_classify_tier()`
   - Phase 2: For each tier with VMs, call `_bfd_place()` with tier-specific params:
     - HOT: `name_prefix="DS_HOT"`, `max_vms_override=10`
     - WARM: `name_prefix="DS_WARM"`, standard constraints
     - COLD: `name_prefix="DS_COLD"`, standard constraints
   - Each tier gets INDEPENDENT bins (fresh _bfd_place call — anti-affinity is natural)
   - Combine all builders (isolated + hot + warm + cold), convert to recommendations
   - Compute metrics, return LayoutProposal with strategy_name="performance"

4. **`_uniform_strategy(vms, constraints) -> LayoutProposal`:**
   - If no VMs, return proposal with empty datastores
   - Compute DS count: `max(ceil(total_cap/usable), ceil(total_iops/budget), 1)`
   - Create `ds_count` builders all named DS_UNIFORM_NN
   - LPT: sort VMs descending by `required_mib`, assign each to least-loaded bin (`min(bins, key=lambda b: b.used_capacity_mib)`)
   - Convert to recommendations, compute metrics
   - Return LayoutProposal with strategy_name="uniform"

5. **`generate_all_proposals(summary, constraints=None) -> list[LayoutProposal]`:**
   - Public entry point (add to `__all__`)
   - Default constraints if None
   - Extract `vms = list(summary.vm_calculations)`
   - If `not summary.has_performance_data`: apply `_apply_default_iops()` to all VMs
   - If `not vms`: return 3 empty proposals (one per strategy)
   - Call all 3 strategies, return list of 3 proposals
   - Order: [consolidation, performance, uniform]

Important:
- Anti-affinity is naturally enforced because each tier gets its own `_bfd_place()` call with independent bins — do NOT add explicit anti-affinity checks inside `_bfd_place`
- Each strategy is a pure function — no shared mutable state between calls
- Use StrEnum for PerformanceTier (project uses StrEnum in config.py already)
