Create `layout_models.py` with 4 frozen dataclasses following the existing pattern in `calculation.py`:

1. **PlacementConstraints** (frozen):
   - `max_ds_capacity_mib: float = 4 * 1024 * 1024` (4 TiB)
   - `max_vms_per_ds: int = 25`
   - `iops_budget_per_ds: float = 100_000.0`
   - `snapshot_reserve_pct: float = 15.0`
   - `growth_margin_pct: float = 20.0`
   - Properties: `usable_ratio` and `usable_capacity_mib`

2. **DatastoreRecommendation** (frozen):
   - `name: str`, `raw_capacity_mib: float`, `usable_capacity_mib: float`
   - `assigned_vms: tuple[VMCalculation, ...]` (use tuple, NOT list)
   - `used_capacity_mib: float`, `utilization_pct: float`, `total_iops: float`, `vm_count: int`
   - `workload_types: frozenset[str] = field(default_factory=frozenset)`

3. **LayoutMetrics** (frozen):
   - All fields from research Pattern 6: total_ds_count, capacities, utilization stats, VM density, IOPS stats, isolation_score, snapshot_granularity_rating, oversized_vm_count

4. **LayoutProposal** (frozen):
   - `strategy_name: str` ("consolidation" | "performance" | "uniform")
   - `datastores: tuple[DatastoreRecommendation, ...]`
   - `metrics: LayoutMetrics`

Also add `DEFAULT_IOPS_BY_WORKLOAD` dict and `_DEFAULT_IOPS_FALLBACK = 50.0` constant in layout_models.py (see research Pattern 7 for exact keys from DRR.csv).

Create `layout_engine.py` with:

1. **`_DatastoreBuilder`** (mutable, NOT frozen) — internal builder during placement:
   - `can_fit(vm, max_vms, iops_budget) -> bool` checks all 3 constraints
   - `add_vm(vm)` updates state
   - `to_recommendation() -> DatastoreRecommendation` freezes output

2. **`_bfd_place(vms, constraints, name_prefix, max_vms_override=None) -> list[_DatastoreBuilder]`**:
   - Sort VMs by `max(capacity_ratio, iops_ratio)` descending
   - For each VM: find bin with least remaining capacity that still fits; if none, open new bin
   - Handle oversized VMs: before BFD loop, separate VMs where `required_mib > usable_capacity_mib` into dedicated datastores named `{name_prefix}_OVER_{idx:02d}`

3. **`_apply_default_iops(vm: VMCalculation) -> VMCalculation`**:
   - When `vm.peak_iops == 0`, use `dataclasses.replace(vm, peak_iops=estimated, avg_iops=estimated*0.7)` where estimated comes from `DEFAULT_IOPS_BY_WORKLOAD.get(vm.workload_category, _DEFAULT_IOPS_FALLBACK)`
   - Use `dataclasses.replace()` since VMCalculation is frozen (do NOT construct manually)

4. **`_compute_metrics(datastores: list[DatastoreRecommendation]) -> LayoutMetrics`**:
   - Handle empty list (return zeroed LayoutMetrics)
   - Compute all fields per research Pattern 6
   - Count oversized VMs (those in `*_OVER_*` named datastores)

5. **`_consolidation_strategy(vms, constraints) -> LayoutProposal`**:
   - Call `_bfd_place(vms, constraints, "DS_CONSOL")`
   - Convert builders to recommendations
   - Compute metrics
   - Return `LayoutProposal(strategy_name="consolidation", ...)`

Important:
- Never import from `ui/` — this is a pipeline module
- Use `from __future__ import annotations` at top of both files
- Add `__all__` exports in both files
- Use `dataclasses.replace()` for modifying frozen dataclasses, not manual construction
