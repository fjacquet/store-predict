  <name>Task 1: Create session_archive.py with save and restore functions</name>
  <files>src/store_predict/pipeline/session_archive.py</files>
  <action>
Create `src/store_predict/pipeline/session_archive.py` with the following public API:

**Module docstring:** "Save and restore a complete StorePredict sizing session as a portable .zip archive."

**Sentinel constant:**
```python
SESSION_ZIP_SENTINEL = "session.json"
```
This file's presence at the root of a .zip identifies it as a StorePredict session archive (not a LiveOptics .zip, which never contains session.json).

**`save_session_zip(session_data: dict[str, object], original_file_bytes: bytes, original_filename: str) -> bytes`**

Build and return the raw bytes of a .zip archive containing:
1. The original uploaded file stored at its original_filename path inside the zip.
2. A `session.json` at the zip root containing a JSON object with:
   - `"schema_version": 1` — for forward-compat
   - `"original_filename": original_filename` — so restore knows the format
   - `"vm_data"` — list of VM record dicts (from session_data["vm_data"]), already JSON-serializable (NaN already converted to null in state.py)
   - `"project_name"` — str
   - `"storage_model"` — str (e.g. "powerstore")
   - `"selected_datacenters"` — list[str]
   - `"selected_clusters"` — list[str]
   - `"layout"` — dict with keys: max_ds_mib, max_vms, iops_budget, snapshot_pct, growth_pct (all float/int)
   - `"compute"` — dict with keys: preset, overcommit, vmsc, ap, custom_cps, custom_sockets, custom_ram, vmsc_split, ap_active (mixed types matching app.storage.tab values)

Extract each config value from session_data using the same keys that the UI pages use:
- layout keys: `layout_max_ds_mib`, `layout_max_vms`, `layout_iops_budget`, `layout_snapshot_pct`, `layout_growth_pct`
- compute keys: `compute_preset`, `compute_overcommit`, `compute_vmsc`, `compute_ap`, `compute_custom_cps`, `compute_custom_sockets`, `compute_custom_ram`, `compute_vmsc_split`, `compute_ap_active`

Use `zipfile.ZipFile` with `compression=zipfile.ZIP_DEFLATED` and write to an `io.BytesIO` buffer. Return `buf.getvalue()`.

**`is_session_zip(content: bytes) -> bool`**

Return True if content is a valid ZIP and contains SESSION_ZIP_SENTINEL at root. Use `zipfile.ZipFile(io.BytesIO(content)).namelist()` and check membership. Return False on any exception (bad zip etc.).

**`restore_session_zip(content: bytes) -> dict[str, object]`**

Read the .zip, parse session.json, and return a flat dict of app.storage.tab keys ready to be written:
- `"vm_data"` → list from snapshot
- `"project_name"` → str from snapshot
- `"storage_model"` → str from snapshot
- `"selected_datacenters"` → list from snapshot
- `"selected_clusters"` → list from snapshot
- `"layout_max_ds_mib"` → float(snapshot["layout"]["max_ds_mib"])
- `"layout_max_vms"` → int(snapshot["layout"]["max_vms"])
- `"layout_iops_budget"` → float(snapshot["layout"]["iops_budget"])
- `"layout_snapshot_pct"` → float(snapshot["layout"]["snapshot_pct"])
- `"layout_growth_pct"` → float(snapshot["layout"]["growth_pct"])
- `"compute_preset"` → str
- `"compute_overcommit"` → float
- `"compute_vmsc"` → bool
- `"compute_ap"` → bool
- `"compute_custom_cps"` → int
- `"compute_custom_sockets"` → int
- `"compute_custom_ram"` → int
- `"compute_vmsc_split"` → int
- `"compute_ap_active"` → int
- `"_restored_original_filename"` → str (the original_filename from JSON, used by upload page to set the format hint)
- `"_restored_original_bytes"` → bytes (raw bytes of the original file extracted from the zip)

Raise `IngestionError` (from `store_predict.pipeline.errors`) if: the zip is invalid, session.json is missing, JSON cannot be parsed, or schema_version != 1.

Use `from __future__ import annotations`, strict typing with mypy-compatible annotations. Import `io`, `json`, `zipfile` from stdlib only (no pandas dependency — this module is pure stdlib + one internal import).
