<objective>
Fix the AG Grid row identity bug by switching getRowId from vm_name to a stable integer row_index column. This is a required foundation change before any further grid work — duplicate VM names in customer RVTools exports cause AG Grid to collapse rows, making inline edits apply to the wrong VM.

Purpose: Eliminate row identity corruption for customer files with duplicate VM names (linked clones, template copies). Without this fix, the column visibility features planned in 20-02 would be built on a broken foundation.
Output: row_index registered in CANONICAL_COLUMNS, assigned in ingestion.py, and used as getRowId in the grid with both cell-change handlers updated atomically.
</objective>

In `src/store_predict/pipeline/ingestion.py`, after the line:

```python
df = df[~df["is_template"]].reset_index(drop=True)
```

add:

```python
df["row_index"] = df.index.astype(int)
```

This assigns a contiguous 0..N-1 integer after template VMs have been filtered out. Using `.astype(int)` guards against float64 inference if the DataFrame is later serialized to JSON and reconstructed.

Do NOT assign row_index inside the individual parsers (rvtools.py, liveoptics.py). The parsers produce raw DataFrames with gaps in the index after any internal filtering. Assigning in `ingest_file` (post-filter, post-reset_index) guarantees contiguous unique values. The parsers do not need a placeholder column because the CANONICAL_COLUMNS filter runs inside the parsers — but `row_index` must be in CANONICAL_COLUMNS so it is not stripped. Wait: the parsers return `result[CANONICAL_COLUMNS]` which would drop row_index if it is not assigned before that line. Check rvtools.py and liveoptics.py — if they end with `return result[CANONICAL_COLUMNS]`, add `result["row_index"] = 0` (placeholder integer) before that return in each parser so the column is preserved through the whitelist filter. The real value is then overwritten by `ingest_file` after template filtering and `reset_index`.

Specifically in `src/store_predict/pipeline/parsers/rvtools.py`, find the `return result[CANONICAL_COLUMNS]` line and add `result["row_index"] = 0` immediately before it.
In `src/store_predict/pipeline/parsers/liveoptics.py`, find all `return result[CANONICAL_COLUMNS]` lines (there may be multiple — one for xlsx, one for csv) and add `result["row_index"] = 0` before each.

This two-step approach (placeholder in parsers, real value in ingest_file) is the correct pattern per research findings.

Change the getRowId grid option from:


The `String(...)` wrapper is required — AG Grid's getRowId callback must return a string. Integer row_index values would be coerced but String() makes it explicit and safe.
