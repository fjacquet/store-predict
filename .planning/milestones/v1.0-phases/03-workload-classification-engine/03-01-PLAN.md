Create `src/store_predict/pipeline/classification.py` with the following components:

1. **Helper functions:**
   - `_patterns(*keywords: str) -> tuple[re.Pattern[str], ...]` — compile case-insensitive literal substring patterns using `re.escape()`
   - `_regex_patterns(*expressions: str) -> tuple[re.Pattern[str], ...]` — compile case-insensitive regex patterns (for alternation like "linux|ubuntu|centos")

2. **ClassificationRule dataclass (frozen=True):**
   - Fields: name (str), category (str), subcategory (str), priority (int), vm_name_patterns (tuple of re.Pattern, default=()), os_patterns (tuple of re.Pattern, default=()), match_mode (str, default="any")
   - `matches(vm_name: str, os_name: str) -> bool` method: "any" mode = vm_name OR os match; "all" mode = both required. Only check pattern types that are defined.

3. **ClassificationResult dataclass (frozen=True):**
   - Fields: category (str), subcategory (str), rule_name (str), confidence (str — "rule_match" | "os_fallback" | "default")

4. **RuleRegistry class:**
   - `__init__(rules: list[ClassificationRule])` — sort by priority ascending
   - `classify(vm_name: str, os_name: str) -> ClassificationResult` — iterate rules in priority order, first match wins. Confidence derived from priority: <900 = "rule_match", 900-998 = "os_fallback", 999 = "default". Fallback if no rule matches: Unknown (Reducible).
