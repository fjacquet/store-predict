**workload_dialog.py:**

Create `WorkloadDialog(ui.dialog)` subclass following the awaitable dialog pattern from RESEARCH.md:

```python
class WorkloadDialog(ui.dialog):
    def __init__(
        self,
        vm_name: str,
        current_workloads: list[str],
        all_options: list[dict],
    ) -> None:
```

Parameters:

- `vm_name`: displayed in dialog title
- `current_workloads`: list of currently selected workload labels (e.g., ["Database / Microsoft SQL"])
- `all_options`: list of dicts with keys `label`, `value` (where value = label string for simplicity)

Dialog layout:

- `ui.card().classes("w-96")` inside `self`
- Title: `f"Workloads for {vm_name}"` (text-lg font-bold)
- Subtitle: "Select one or more workload types. Conservative (lowest) DRR will be used." (text-sm text-gray-500)
- `ui.select` with `multiple=True`, `options=all_options`, `value=current_workloads`, `.props("use-chips")` (prevents dialog close on select per Pitfall 4), `.classes("w-full")`
- Button row (justify-end):
  - Cancel button: `on_click=lambda: self.submit(None)`
  - Apply button: `on_click=lambda: self.submit(self.select.value)`, blue styling

Add `.props("persistent")` to the dialog itself to prevent backdrop-click closing during multi-select interaction.

Store `self.select` as instance attribute for the submit lambda to reference.

- Value: `text-2xl font-bold`
