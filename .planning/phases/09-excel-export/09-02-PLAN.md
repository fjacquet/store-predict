    3. Add the `_on_download_excel` handler function after the existing `_on_download` function at the bottom of the file:
       ```python
       def _on_download_excel(summary: object, project_name: str) -> None:
           """Generate Excel workbook and trigger browser download."""
           from store_predict.pipeline.calculation import CalculationSummary

           assert isinstance(summary, CalculationSummary)
           xlsx_bytes = generate_report_xlsx(summary, project_name, locale=get_locale())
           safe_name = sanitize_filename(project_name)
           date_str = datetime.now(tz=UTC).strftime("%Y-%m-%d")
           filename = f"StorePredict_{safe_name}_{date_str}.xlsx"
           ui.download(
               xlsx_bytes,
               filename=filename,
               media_type="application/vnd.openxmlformats-officedocument.spreadsheetml.sheet",
           )
       ```

    Implementation notes:
    - `sanitize_filename` and `get_locale` are already imported in report.py — reuse them.
    - `datetime` and `UTC` are already imported — reuse them.
    - The `generate_report_xlsx` import goes at the module top-level (not lazy) since the module is only loaded server-side.
    - Do NOT use `lambda summary=summary` capture hack — `_on_download_excel(summary, project_name)` captures by reference in the lambda, which is correct here because `summary` is a local variable in `report_page()` that does not change after creation.
