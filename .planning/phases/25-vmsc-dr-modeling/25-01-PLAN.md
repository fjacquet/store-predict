  <action>
Make the following changes to compute_sizing.py:

1. Replace vmsc_hosts_per_site (single int) with vmsc_site_a_hosts and vmsc_site_b_hosts (two ints) in ComputeSizingResult. Both default to 0. Remove vmsc_hosts_per_site entirely.

2. Add two new parameters to compute_sizing():
   - vmsc_split_ratio: float = 0.5 — fraction of VMs on Site A (Site B gets 1 - vmsc_split_ratio). Clamped to [0.01, 0.99] internally.
   - ap_active_ratio: float = 1.0 — fraction of VMs active on primary AP site. Clamped to [0.01, 1.0] internally.

3. Update _empty_result() to use vmsc_site_a_hosts=0, vmsc_site_b_hosts=0 (remove vmsc_hosts_per_site).

4. vMSC per-site sizing logic (replaces current vmsc_hosts calculation):
   When vmsc_enabled=True and vmsc_avail=True:
   - site_a_vcpus = round(total_vcpus * clamped_vmsc_split_ratio)
   - site_b_vcpus = total_vcpus - site_a_vcpus
   - site_a_ram_gib = total_ram_gib * clamped_vmsc_split_ratio
   - site_b_ram_gib = total_ram_gib - site_a_ram_gib
   - site_a_hosts = max(_hosts_n1(site_a_vcpus, host_pcores, ratio), _hosts_by_ram(site_a_ram_gib, host_config.ram_gib))
   - site_b_hosts = max(_hosts_n1(site_b_vcpus, host_pcores, ratio), _hosts_by_ram(site_b_ram_gib, host_config.ram_gib))
   Otherwise: site_a_hosts = site_b_hosts = 0

5. A/P DR sizing logic (replaces current ap_primary/ap_secondary calculation):
   - active_vcpus_primary = round(total_vcpus * clamped_ap_active_ratio)
   - active_ram_gib_primary = total_ram_gib * clamped_ap_active_ratio
   - ap_primary = max(_hosts_n1(active_vcpus_primary, host_pcores, ratio), _hosts_by_ram(active_ram_gib_primary, host_config.ram_gib))
   - ap_secondary = max(1, math.ceil(ap_primary / 2))  — secondary remains at 50% of primary (cold standby)

   Note: This preserves the existing ap_primary_hosts / ap_secondary_hosts field names; only the calculation changes to use ap_active_ratio.

6. Update __all__ to reflect removed vmsc_hosts_per_site if it was exported.

7. Add vmsc_split_ratio and ap_active_ratio to the function signature docstring. Document clamping behavior. Note that ap_secondary remains at 50% of computed primary (cold standby convention per pre-sales).

Keep all existing logic for edge cases (empty df, no datacenter column, all VMs powered off, etc.).
