Improve classification accuracy by stripping configurable company name prefixes from VM names and using the vm_description field as an additional matching signal.

Purpose: Many environments prefix all VMs with company name (e.g. "ACME-SQLPROD01"), hiding the workload keyword. Description fields often contain useful workload hints too.

Output: Enhanced classification engine that strips prefixes and checks description, plus targeted tests.
</objective>

  <files>
    src/store_predict/pipeline/classification.py
  </files>
  <action>
    1. Add new function strip_company_prefix(vm_name: str, patterns: list[str]) -> str:
       - For each pattern in patterns, apply re.sub(pattern, "", vm_name, count=1)
       - Return the stripped name (or original if no pattern matched)
       - Patterns are regex strings anchored to start (e.g. r"^ACME[-_]")
       - Use re.IGNORECASE

    2. Modify ClassificationRule.matches to accept optional description parameter:
       - New signature: matches(self, vm_name: str, os_name: str, description: str = "") -> bool
       - If no vm_name_patterns match AND description is non-empty, also check vm_name_patterns against description
       - This means description acts as a fallback signal for vm_name patterns only
       - os_patterns still only match against os_name (description is not an OS field)
       - This approach ensures description matching only supplements, never overrides

    3. Modify RuleRegistry.classify to accept optional description parameter:
       - New signature: classify(self, vm_name: str, os_name: str, description: str = "") -> ClassificationResult
       - Before matching, call strip_company_prefix on vm_name using COMPANY_PREFIX_PATTERNS from config
       - Pass description to rule.matches()
       - Import COMPANY_PREFIX_PATTERNS from config at top of file

    4. Modify classify_dataframe to:
       - Read "vm_description" column from df if it exists, default to ""
       - Pass description to registry.classify() for each row
       - Use: description = str(row["vm_description"]) if "vm_description" in df.columns and pd.notna(row.get("vm_description")) else ""
  </action>
