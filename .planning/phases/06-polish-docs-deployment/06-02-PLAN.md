  <name>Task 1: Create server-side file validation module and integrate into upload</name>
  <files>src/store_predict/pipeline/validation.py, src/store_predict/ui/pages/upload.py, tests/test_validation.py</files>
  <action>
Create `src/store_predict/pipeline/validation.py` with a `validate_upload(content: bytes, filename: str) -> None` function:
- Check file extension is `.xlsx` or `.csv` (case-insensitive). Raise `IngestionError` if not.
- For `.xlsx` files: check first 4 bytes are `PK\x03\x04` (ZIP magic). Raise `IngestionError("File does not appear to be a valid .xlsx file")` if not.
- For `.csv` files: try decoding first 1024 bytes as UTF-8. Raise `IngestionError("File does not appear to be a valid CSV file")` if `UnicodeDecodeError`.
- Import `IngestionError` from `store_predict.pipeline.errors`.

In `src/store_predict/ui/pages/upload.py`, in `_handle_upload`:

- After `content = await e.file.read()` and before writing to temp file, call:

  ```python
  from store_predict.pipeline.validation import validate_upload
  validate_upload(content, e.file.name)
  ```

  Move the import to the top of the file.

Create `tests/test_validation.py`:

- Test valid xlsx bytes (starts with `PK\x03\x04`) passes.
- Test valid csv bytes (UTF-8 text) passes.
- Test `.exe` extension raises `IngestionError`.
- Test `.xlsx` with wrong magic bytes raises `IngestionError`.
- Test `.csv` with binary content raises `IngestionError`.
- Use real bytes, not mocks. Create small byte literals for test fixtures.
  </action>
  <verify>Run `rtk pytest tests/test_validation.py -v`. All tests pass. Run `rtk ruff check src/store_predict/pipeline/validation.py tests/test_validation.py`.</verify>
  <done>Server-side validation rejects non-xlsx/csv files by magic bytes and extension. Upload handler calls validate_upload before pipeline processing.</done>
</task>
